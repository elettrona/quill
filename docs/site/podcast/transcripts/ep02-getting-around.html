Liam: Welcome back to The QUILL Cast. I'm Liam.

Jessica: And I'm Jessica. Today is all about movement: how you get anywhere in QUILL without touching a mouse, and how you bend the keyboard to your will.

Liam: Let's start with the single most important keystroke in the application. Control shift P. The command palette.

Jessica: The palette is a searchable list of every registered command in QUILL, and there are hundreds. Type any part of a command's name to filter the list, arrow through the matches, press enter to run one. Want to spell check? Type S P E L L. Want to compare documents? Type C O M P. You never need to know where a command lives in the menus.

Liam: And here's the part people miss: the palette shows each command's current keyboard shortcut right next to its name. So the palette is also how you learn the shortcuts. Use a command through the palette three or four times, notice the key next to it, and eventually you just press the key.

Jessica: It searches both directions, too. In the Keymap Editor you can type an actual shortcut, like control alt M, and QUILL tells you which command owns that key, or that it's free. No more guessing whether a key is taken.

Liam: Next concept: the QUILL key. QUILL has more commands than there are convenient key combinations, so it borrows an idea from the great keyboard programs of history: a prefix key. By default it's control shift grave, the key with the tilde on it.

Jessica: You press the QUILL key once, and the next key you press is interpreted as a command. QUILL key then S inserts a snippet. QUILL key then Shift O opens a file from a remote server. It's a two-step dance: prefix, then letter.

Liam: And if you press the QUILL key twice, you lock in what's called Browse Mode, where letters and arrows keep acting as navigation commands until you press escape. It's very similar to the virtual cursor idea screen reader users already know from the web.

Jessica: Quick Nav gives you fast document movement: jump by heading in Markdown files, hop between bookmarks, move by section. For long documents it turns scrolling into stepping.

Liam: Let's talk about making the keyboard yours, because QUILL is unusually generous here. Preferences, Keyboard, opens the Keymap Editor. Every command can be bound, rebound, or unbound.

Jessica: The editor is forgiving in all the right ways. You can type modifiers in any order and any spelling. Control, C T R L, even C T L. It normalizes what you type. Or choose Record Keys and literally press the combination you want.

Liam: And when you pick a key that's already taken, QUILL doesn't silently refuse or silently clobber. It names the command that owns the key, by its friendly title, and offers to move the key to your new command. One step, fully informed.

Jessica: There's also Run Diagnostics in the keymap editor, which audits your whole layout: duplicate shortcuts, bindings pointing at commands that no longer exist, keys that are assigned but can't actually fire. And it offers a one-click heal. Your keymap can be exported as a file and shared, too. They're called keyboard packs, and the community trades them.

Liam: Now the main window itself. It's a deliberately simple shape: a menu bar, a tabbed editor area, and a status bar. But both bars are smarter than they look.

Jessica: The status bar is your quiet narrator. It shows and optionally speaks what just happened: file saved, twelve results found, GLOW audit opened. And the key thing is you control the chattiness. QUILL has an entire verbosity system, with profiles from near-silent to fully narrated, quiet hours, and even a command that answers, why did QUILL say that?

Liam: My favorite piece of that system is the Spoken Echo. Speech is ephemeral. Your screen reader says something, it's gone. Alt shift E opens a list of the last twenty things QUILL announced, newest first, and you can arrow through them, re-read them, copy them. Missed an announcement while typing? It's right there.

Jessica: And one more navigation gem: Where am I? QUILL has status query commands that tell you your position, what changed recently, and what context you're in, on demand. Instead of QUILL deciding what to tell you, you ask when you want to know.

Liam: Let's close with a workout. Five things to try today. One: press control shift P and just read the palette for a minute. Arrow through it. See what exists.

Jessica: Two: search the palette for the word GLOW, and run GLOW Audit Current Document on anything you're writing. We'll cover GLOW properly in episode nine, but the report you get is a nice preview.

Liam: Three: open Preferences, Keyboard, and rebind one command you use constantly to a key you love. Just one. Feel the power.

Jessica: Four: press alt shift E after a few minutes of editing and review your Spoken Echo. Five: press F1 in some dialog, any dialog, and see the context help.

Liam: Next episode: documents and formats. Word files, Markdown, EPUB, PDFs, rich formatting without visual clutter, and the trick QUILL uses to keep formatting inside a plain text file.
