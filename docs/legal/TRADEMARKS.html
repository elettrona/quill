Trademark and Naming Guidelines

This document explains how the QUILL for All project name and related branding should be used.

Project Name

The preferred public name of the project is:

QUILL for All

The preferred full organisational form is:

QUILL for All by Community Access

After the full name has been used once in a document or page, the short name QUILL may be used when the context is clear.

Purpose of These Guidelines

These guidelines are intended to:

  • Reduce confusion with other products, projects, or organisations that use a similar name.
  • Keep the project’s identity consistent.
  • Help contributors use the project name clearly and respectfully.
  • Preserve the accessibility-focused mission of the project.

Correct Usage

Preferred:

  • QUILL for All
  • QUILL for All by Community Access
  • Community Access QUILL, when referring to the Community Access project specifically

Acceptable short form after first use:

  • QUILL

Avoid using bare QUILL as the only project name in public-facing materials where confusion may occur.

Description

Preferred one-line description:

QUILL for All is an open-source, accessibility-focused editor from Community Access.

Longer description:

QUILL for All helps people write, edit, convert, compare, and publish documents in a screen-reader-friendly environment.

Independence Notice

QUILL for All is an independent open-source project by Community Access. It is not affiliated with, sponsored by, or endorsed by Quill.js, QuillBot, Quill.org, or any other similarly named product, project, company, or organisation.

Third-Party Marks

All third-party trademarks, service marks, product names, company names, and logos are the property of their respective owners.

References to third-party names are for identification, compatibility, attribution, documentation, or comparison purposes only.

Contributor Guidance

Contributors should:

  • Use QUILL for All in user-facing documentation, release notes, installers, websites, and announcements.
  • Avoid creating logos, icons, or taglines that imply affiliation with other similarly named products.
  • Avoid using third-party names in a way that suggests endorsement, sponsorship, or partnership unless such a relationship has been approved in writing.
  • Preserve legal notices and attribution files when redistributing the project.

Forks and Modified Versions

Forks and modified versions should avoid representing themselves as official releases of QUILL for All unless they are published by the official project maintainers.

Forks should use naming that clearly distinguishes them from the official Community Access project.

Examples:

  • “Example Editor, based on QUILL for All”
  • “Example QUILL fork”
  • “Example Accessibility Editor using QUILL for All components”

Avoid names that imply official status without permission.

Questions

Questions about project naming, branding, or trademark use should be directed to the Community Access maintainers.
