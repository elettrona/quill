# QUILL Speech Project Profile Format

**File:** `<project>/.quill/speech-project.json`
**Schema:** `quill/core/schemas/speech_project.json` (JSON Schema 2020-12)
**Code:** `quill/core/speech/project_profile.py`
**Related:** §4.7, §4.8, §4.9, §4.10 of [batch-document-to-speech-plan.md](batch-document-to-speech-plan.md)
**Status:** Implemented (core); UI surface pending
**Date:** 2026-06-24

---

## 1. What this is

A *project* in QUILL is a folder of documents (Word, Markdown, HTML, plain text).
The **speech project profile** is a small JSON file stored inside that folder, at
`.quill/speech-project.json`, that remembers the whole speech configuration for
the folder: which synthesizer and voice to use, the output format, how to
chapterize by heading, how to clean text for speech, and which pronunciation
dictionaries (and in what format) are active.

The goal (§4.10) is **configure once per project, never re-enter it**. When a
project opens, QUILL loads this file and applies it. The file travels with the
folder, so moving or sharing the folder carries its speech settings along.

This format is deliberately small, human-readable, and forward-tolerant: unknown
keys are ignored, missing keys fall back to recommended defaults, and the
`version` field allows future migrations.

## 2. Location and lifecycle

- **Path:** `profile_path(project_dir)` returns `<project_dir>/.quill/speech-project.json`.
- **Write:** `save_profile(profile, project_dir)` writes it atomically (temp file
  plus `os.replace`), creating `.quill/` if needed.
- **Read:** `load_profile(project_dir)` returns a `SpeechProjectProfile`, or
  `None` when the folder has no profile yet (so callers can apply global defaults).
- **Precedence (planned, §4.10):** this-run options > project profile > global
  settings > built-in defaults. The profile is the *project* tier.

The same `.quill/` folder also holds project-scope pronunciation dictionaries
(`.quill/pronunciation/<id>.json`, see §4.7), which this profile references.

## 3. Top-level shape

```json
{
  "version": 1,
  "synthesizer": { "engine": "sapi5", "voice": "", "rate": 200, "volume": 1.0, "speed": 1.0, "extra": {} },
  "output": { "format": "wav", "skip_existing": false },
  "chapters": {
    "mode": "none",
    "sound_enabled": false,
    "sound_id": "",
    "sound_volume": 100,
    "article_gap_ms": 1200,
    "intro_section_title": "Introduction"
  },
  "normalization": {},
  "pronunciation": { "enabled": true, "dictionaries": [] }
}
```

### 3.1 `synthesizer`

The TTS engine and its voice/pacing. Local engines only (cloud engines are
excluded from batch, §2.3).

| Key | Type | Notes |
|-----|------|-------|
| `engine` | enum | `sapi5`, `dectalk`, `piper`, `kokoro`, `espeak`. |
| `voice` | string | Engine voice id/name (a SAPI voice token, a Kokoro voice, an eSpeak language, a Piper model name). |
| `rate` | int 50-800 | Words-per-minute style rate (SAPI 5 / eSpeak / DECtalk). |
| `volume` | number 0.0-1.0 | Output volume. |
| `speed` | number 0.25-4.0 | Speed multiplier for engines that use one (Kokoro). |
| `extra` | object | Engine-specific keys (see below). |

Recognised `extra` keys, by engine:

- **piper:** `piper_executable`, `piper_model` (paths).
- **dectalk:** `dectalk_executable`, `dectalk_dictionary` (paths).
- **espeak:** `espeak_executable` (path).

### 3.2 `output`

| Key | Type | Notes |
|-----|------|-------|
| `format` | enum | `wav` (always available) or `mp3` (requires ffmpeg; falls back to WAV with a per-file note). |
| `skip_existing` | bool | Skip a file whose output already exists, so a re-run resumes cheaply. |

### 3.3 `chapters`

Heading-aware chapterization (§4.8). Applies when a document has headings (Word /
Markdown / HTML; plain text has none and is one section).

| Key | Type | Notes |
|-----|------|-------|
| `mode` | enum | `none` (one flat file), `single` (one file with MP3 chapter markers), `separate` (one file per article). |
| `sound_enabled` | bool | Play a transition earcon at each boundary; also emits a *with-tones* file variant alongside the clean one. |
| `sound_id` | string | Sound-pack sound id for the earcon. `""` = the bundled placeholder chime (a short ding-dong generated by `quill/core/speech/earcon.py`, replaceable by the user). |
| `sound_volume` | int 0-100 | Earcon volume. |
| `article_gap_ms` | int 0-10000 | Silence inserted between articles, in milliseconds. |
| `intro_section_title` | string | Title for the lead-in section before the first heading, and for documents with no headings. |

Chapter titles themselves come from the document's headings; this profile only
controls the lead-in title, the gap, and the earcon.

### 3.4 `normalization`

A serialized `TextNormalizationOptions` object (§4.9). An empty object means
"recommended defaults". The full set of passes (quotes, dashes, ellipsis,
invisibles, ligatures, symbols, fractions, emoji, phones/emails/URLs, the user
replacement map, etc.) is documented in `quill/core/speech/text_normalize.py`.

### 3.5 `pronunciation`

Which pronunciation dictionaries are active for the project (§4.7), and in what
format.

| Key | Type | Notes |
|-----|------|-------|
| `enabled` | bool | Master toggle for pronunciation correction in this project. |
| `dictionaries` | array | Ordered references; earlier entries are more specific on conflict (the resolver also applies scope/engine precedence). |

Each dictionary reference:

| Key | Type | Notes |
|-----|------|-------|
| `id` | string (required) | Dictionary id; for `quill-json` this is the JSON file stem. |
| `scope` | enum | `global` (app data dir) or `project` (the project's `.quill/pronunciation/`). |
| `format` | enum | `quill-json` (native), or the reserved importers `sapi-lexicon`, `csv`. |
| `path` | string | For `project` scope: path relative to the project root. `""` = resolve by id in the standard location. |

## 4. A worked example

A "Daily News" folder of Word articles, read by Piper, exported as chaptered MP3
with a transition chime and a 1.5 s pause between articles, using a project-scoped
news pronunciation dictionary plus a global tech-terms dictionary:

```json
{
  "version": 1,
  "synthesizer": {
    "engine": "piper",
    "voice": "en_US-amy-medium",
    "rate": 200,
    "volume": 1.0,
    "speed": 1.0,
    "extra": {
      "piper_executable": "C:/Tools/piper/piper.exe",
      "piper_model": "C:/Tools/piper/models/en_US-amy-medium.onnx"
    }
  },
  "output": { "format": "mp3", "skip_existing": true },
  "chapters": {
    "mode": "single",
    "sound_enabled": true,
    "sound_id": "",
    "sound_volume": 80,
    "article_gap_ms": 1500,
    "intro_section_title": "Front Page"
  },
  "normalization": { "dash_mode": "words", "speak_urls": true },
  "pronunciation": {
    "enabled": true,
    "dictionaries": [
      { "id": "daily-news", "scope": "project", "format": "quill-json", "path": ".quill/pronunciation/daily-news.json" },
      { "id": "tech-terms", "scope": "global", "format": "quill-json", "path": "" }
    ]
  }
}
```

## 5. Driving the pipeline from a profile (headless)

`project_profile.py` provides converters so the batch pipeline can run straight
from a profile, with no UI:

- `to_batch_options(profile, source_folder, output_folder, pronunciation_dictionaries=...)`
  builds a `BatchExportOptions` (engine, voice, rate/speed, output format,
  skip-existing, dictionaries).
- `to_chapter_options(profile)` builds a `ChapterAssembleOptions` (gap, earcon,
  intro title, output format) for `chapter_assemble.assemble_chaptered_audio`.

This is what the headless end-to-end tests use to stress every setting/option
combination before any UI exists.

## 6. Validation rules (summary)

All values are clamped/normalised on load (`from_dict`), never rejected:

- `synthesizer.engine` falls back to `sapi5` if unknown; `rate` clamps to 50-800,
  `volume` to 0.0-1.0, `speed` to 0.25-4.0.
- `output.format` falls back to `wav` if unknown.
- `chapters.mode` falls back to `none`; `sound_volume` clamps to 0-100;
  `article_gap_ms` clamps to 0-10000; a blank `intro_section_title` falls back to
  `Introduction`.
- `pronunciation.dictionaries` entries without an `id` are dropped; unknown
  `scope`/`format` fall back to `global` / `quill-json`.
