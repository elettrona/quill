Why the from format has that plus sign. gfm is GitHub-flavored Markdown, and the editor is line-oriented, one editor line is one paragraph. Bare gfm treats a single newline as a soft wrap, so every export would join your lines into one giant paragraph. gfm plus hard line breaks tells Pandoc to honor each newline as a paragraph break. The result is that a bulleted list you wrote line by line stays a bulleted list in Word, in PDF, in EPUB, in OpenDocument. This is the part that quietly breaks a lot of editors. QUILL got it right by default and put a code comment explaining why, so the next maintainer doesn't accidentally drop the plus sign.
